Programmierphiliosophie

Meine Art zu Programmieren leitet sich ebenfalls von meinen drei Kernfragen ab. Ist das Problem klar formuliert, kann dieses möglichst direkt addressiert werden. Getreu nach Albert Einstein "So einfach wie möglich, aber nicht einfacher". Meine Erfahrung hat mich gelehrt, das Software im Laufe der Zeit mit sich wandelnden Anforderungen wie von alleine komplexer wird. Deshalb versuche ich es von Anfang an unnötige Komplexität zu vermeiden.

Ich bin ein großer Fan von Testgetriebener Entwicklung (TDD). Zum einen wird man dadurch gedrängt "des Pudels Kern" zu suchen, zum anderen ermöglicht das Flexibiltät weil es die Grundlage für rigoroses code refactoring darstellt.

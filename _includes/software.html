Programmierphilosophie

Meine Art zu Programmieren leitet sich ebenfalls von meinen drei Kernfragen ab. Ist das Problem klar formuliert? Kann dieses möglichst direkt adressiert werden? Getreu nach Albert Einstein "So einfach wie möglich, aber nicht einfacher". Meine Erfahrung hat mich gelehrt, das Software im Laufe der Zeit, mit sich wandelnden Anforderungen, wie von alleine komplexer wird. Deshalb bemühe ich mich stehts unnötige Komplexität zu vermeiden.

Deneben bin ich ein großer Fan von Testgetriebener Entwicklung (TDD). Zum einen wird man dadurch gedrängt "des Pudels Kern" zu suchen, zum anderen ermöglicht das Flexibiltät, weil es die Grundlage für rigoroses code refactoring darstellt.

TechStack

Unter TechStack verstehe ich Technologien mit denen man eine Menge Zeit verbracht hat, bzw. zumindest soviel, um die dahinter liegenden Konzepte zu verstehen einordnen zu können. Oftmals entstehen Wettbewerbsvorteile heute durch überlegene Technologien. Folglich ist es oft von strategischer Bedeutung, zu wissen, für welche Aufgaben sich welche Technologien/Programmiersprachen besser eignen als andere. Mein Fokus liegt klar auf Python (>> 10.000h), das ich seit 2012 verwende.
  • Programmierparadigmen: skripting, objektorientiert, funktional
  • Softwareentwicklung: Git, Docker, CICD, Kubernetes, Linux, Azure, VIM, Bash
  • Datenbanken: (SQL, NoSQL, GraphQL)
  • Python:
    • numerisch: NumPy, pandas, Pyomo, scikit-learn, Matplotlib, XGBoost, PyMC, Seaborn
    • dev: pytest, Pydantic, SQLAlchemy, Pandera, Poetry, itertools, cytoolz
    • front-end: FastAPI, Django
  • Bekanntschaft mit anderen Programmiersprachen: C, Golang, JavaScript, Julia
